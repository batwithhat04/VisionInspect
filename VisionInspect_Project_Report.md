# VisionInspect - Complete Project Report

## 1. Executive Summary
VisionInspect is a production-grade infrastructure inspection system designed to detect, analyze, and quantify structural damage (such as corrosion, rust, and seepage) using standard smartphones. By deploying an AI-powered pipeline directly on the edge, it offers inspectors real-time analysis, accurate severity grading, and automated report generation without requiring a persistent internet connection.

## 2. Project Objectives
- To automate and digitize structural asset inspection.
- To detect damage in real-time using lightweight edge AI models.
- To precisely calculate the surface area affected by damage (segmentation).
- To eliminate manual data entry through GPS auto-tagging and automated PDF report generation.

## 3. Technical Architecture & Tech Stack

### AI Pipeline (The "Brain")
- **Preprocessing:** CIE Lab Color Space conversion to isolate rust (a* channel) and efflorescence/seepage (b* channel).
- **Object Detection:** YOLOv8 (Nano) optimized for high frame-rate mobile inference.
- **Image Segmentation:** U-Net architecture with a MobileNetV2 backbone for pixel-wise mapping of damaged areas.
- **Quantization:** TensorFlow Lite (TFLite) conversion with INT8/FP16 quantization, vastly reducing model latency and size for edge hardware.

### Mobile Application (The "Edge")
- **Platform:** Android (Kotlin).
- **Architecture Pattern:** MVVM (Model-View-ViewModel) for maintainability.
- **Camera integration:** CameraX for low-latency continuous image streaming.
- **On-Device Database:** Room (SQLite) for offline inspection history storage.
- **Report Generation:** iText7 for generating PDF documents.

## 4. Detailed Workflow

The workflow of VisionInspect ensures a seamless transition from raw camera feed to actionable inspection data:

**Step 1: Real-Time Capture & Preprocessing**
- The inspector opens the app and points the camera at the structural asset.
- CameraX streams frames directly to the `ImagePreprocessor`.
- The frames are enhanced based on real-world lighting conditions (CIE Lab threshold adjustments).

**Step 2: Dual-Model Inference**
- **Detection (YOLOv8):** The frame is passed through the `TFLiteDetector` which quickly draws bounding boxes around suspected damage regions.
- **Segmentation (U-Net):** Once a region of interest is detected, the `TFLiteSegmenter` analyzes the pixels inside the bounding box to generate a precise mask of the damage.

**Step 3: Severity Calculation**
- The app calculates the ratio of damaged pixels (from the U-Net mask) to the total surface area.
- It applies a severity grading:
  - **Minor:** < 5% damage area.
  - **Moderate:** 5% - 20% damage area.
  - **Critical:** > 20% damage area.

**Step 4: Meta-data Tagging**
- The `GPSProvider` automatically fetches coordinates and tags the inspection data with location, date, and time.

**Step 5: Data Persistence & Report Generation**
- Data is saved locally using the `InspectionDao` (Room/SQLite).
- The user can trigger the `PDFReportGenerator` to export the findings.

## 5. Output and Result Format

This section outlines the precise format of the results generated by the VisionInspect system.

### A. Real-Time Application UI (Visual Output)
While inspecting an asset, the user's screen displays:
- **Live Camera Feed:** Overlaid with bounding boxes indicating detected damage.
- **Mask Overlays:** A colored pixel-mask highlighting the exact shape of the seepage or corrosion.
- **Live Metrics Dashboard:**
  - **Damage Type:** (e.g., "Corrosion" or "Seepage").
  - **Confidence Score:** Model's certainty percentage.
  - **Calculated Area:** Damaged area percentage.
  - **Severity Alert:** Color-coded text (Green for Minor, Yellow for Moderate, Red for Critical).

### B. Automated PDF Report Output
When an inspection is completed, the system generates a standardized PDF document (`PDFReportGenerator.kt`). The format includes:
1. **Header:** 
   - Report ID, Date, Time, and Inspector Name/ID.
2. **Location Data:** 
   - GPS Coordinates (Latitude, Longitude).
3. **Inspection Summary:** 
   - Asset condition overview and severity grade (Minor/Moderate/Critical).
4. **Visual Evidence:** 
   - High-resolution snapshot of the original structure.
   - AI-processed image showing detection bounding boxes and segmentation masks.
5. **Detailed Metrics:**
   - Total Surface Area Inspected.
   - Damaged Area Calculated.
   - Percentage of Damage.
6. **Recommendations:** 
   - Automated text based on severity (e.g., "Immediate maintenance required" for Critical severity).

### C. Offline Database Record (Data Output)
All inspections are stored locally via `Room/SQLite` in the following entity format:
- `id`: Primary Key (UUID)
- `timestamp`: Epoch time of inspection
- `latitude`: Float
- `longitude`: Float
- `damage_type`: String (Enum: Rust, Seepage, Crack)
- `severity_grade`: String (Enum: Minor, Moderate, Critical)
- `damage_percentage`: Float
- `image_path`: String (Local URI to original image)
- `processed_image_path`: String (Local URI to masked image)
